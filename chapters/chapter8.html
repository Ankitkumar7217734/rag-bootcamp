Chapter 8: Query Enhancement

Garbage in, garbage out — but for retrieval. Before you ever search the vector store, it pays to fix the question first.

Overview

Every chapter so far has worked on the documents side of RAG — how to chunk them, embed them, store them, and retrieve them well. This chapter flips to the other side of the pipeline: the query itself.

In a RAG pipeline, the quality of the query sent to the retriever determines how good the retrieved context is — and therefore how accurate the LLM's final answer will be. A short, vague, or awkwardly-worded question retrieves weak context no matter how good your vector store is. That's where query enhancement comes in.

What is Query Enhancement?

Query enhancement refers to techniques that improve or reformulate the user's query before retrieval, so the system pulls back better, more relevant documents from the knowledge base.

It's especially useful when:

  • The original query is short, ambiguous, or under-specified.
  • You want to broaden the scope to catch synonyms, related phrases, or spelling variants.
  • There's a language mismatch between how users ask and how documents are written.

This chapter covers the three techniques from the notes, each reshaping the query in a different way:

#TechniqueThe core move
1Query ExpansionRewrite one query into a richer, broader version that catches more relevant docs.
2Query DecompositionBreak one complex question into simple sub-questions, answer each, then combine.
3HyDEGenerate a hypothetical answer with an LLM and embed that to search.
The Core Idea

All three share one principle: better queries → better retrieved chunks → better grounded LLM answers. Where Chapter 7 improved how we search, this chapter improves what we search with.

Key Takeaways

Remember These Four

1. Query enhancement improves retrieval by fixing the question before it reaches the retriever — better queries in, better context out.

2. Query Expansion rewrites one query into a broader, richer version to catch synonyms and related phrasing.

3. Query Decomposition breaks a complex question into sub-questions answered individually, then combined — great for multi-hop and comparison queries.

4. HyDE embeds a hypothetical answer rather than the question, matching documents on answer-style content and bridging vocabulary mismatch.
