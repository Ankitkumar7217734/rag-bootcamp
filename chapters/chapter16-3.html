Hierarchical Agent Teams With RAG

When one supervisor can't manage every worker, compose nested subgraphs — a research team and a writing team, each with its own mid-level supervisor, coordinated by a top-level supervisor.

Overview

The supervisor pattern from 16.2 works well for a handful of specialists. But what happens when the job itself is complex — research and document authoring, each requiring multiple sub-steps? A single supervisor routing among six or eight workers becomes a bottleneck.

The solution is hierarchical agent teams: compose separate subgraphs, each with its own mid-level supervisor, and connect them under a top-level supervisor. The notes describe this as distributing work when “the job for a single worker becomes too complex” or “the number of workers becomes too large.”

Our build implements a research assistant that writes about transformer variants: a Research Team (search + web scraper) gathers information, then a Writing Team (note taker + doc writer + chart generator) produces the final document. A top supervisor picks which team acts next.

Where this fits

Building block 3 of 3 in Chapter 16 — the most complex coordination pattern. It combines everything from 16.1 (ReAct worker nodes) and 16.2 (supervisor routing) into nested graphs: subgraphs compiled independently, then invoked as nodes in a parent graph.

State and the Supervisor Factory

Hierarchical teams extend MessagesState with a next field that tracks the supervisor's routing decision. A reusable factory builds any supervisor node: 

from typing import List, Optional, Literal
from typing_extensions import TypedDict
from langchain_core.language_models.chat_models import BaseChatModel
from langchain_core.messages import HumanMessage
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.types import Command

class State(MessagesState):
    next: str

def make_supervisor_node(llm: BaseChatModel, members: list[str]):
    options = ["FINISH"] + members
    system_prompt = (
        "You are a supervisor tasked with managing a conversation between the"
        f" following workers: {members}. Given the following user request,"
        " respond with the worker to act next. Each worker will perform a"
        " task and respond with their results and status. When finished,"
        " respond with FINISH."
    )

    class Router(TypedDict):
        """Worker to route to next. If no workers needed, route to FINISH."""
        next: Literal[*options]

    def supervisor_node(state: State) -> Command[Literal[*members, "__end__"]]:
        messages = [{"role": "system", "content": system_prompt}] + state["messages"]
        response = llm.with_structured_output(Router).invoke(messages)
        goto = response["next"]
        if goto == "FINISH":
            goto = END
        return Command(goto=goto, update={"next": goto})

    return supervisor_node

Unlike the prebuilt create_supervisor, this factory uses structured output (Router TypedDict) to pick the next worker. The same factory is reused at every level — just pass a different members list.

Research Team Subgraph

The research team has two workers: a search agent (Tavily + internal retriever) and a web scraper agent (fetches full page content from URLs). Each worker is a ReAct subgraph wrapped as a node that always reports back to the team's supervisor:

Tools

from langchain_community.document_loaders import WebBaseLoader
from langchain_tavily import TavilySearch
from langchain_core.tools import tool
from langgraph.prebuilt import create_react_agent

tavily_tool = TavilySearch(max_results=5)

@tool
def scrape_webpages(urls: List[str]) -> str:
    """Use requests and bs4 to scrape the provided web pages for detailed information."""
    loader = WebBaseLoader(urls)
    docs = loader.load()
    return "\n\n".join(
        f'<Document name="{doc.metadata.get("title", "")}">\n'
        f'{doc.page_content}\n</Document>'
        for doc in docs
    )

Worker nodes

search_agent = create_react_agent(llm, tools=[tavily_tool, internal_tool])

def search_node(state: State) -> Command[Literal["supervisor"]]:
    result = search_agent.invoke(state)
    return Command(
        update={"messages": [
            HumanMessage(content=result["messages"][-1].content, name="search")
        ]},
        goto="supervisor",
    )

web_scraper_agent = create_react_agent(llm, tools=[scrape_webpages])

def web_scraper_node(state: State) -> Command[Literal["supervisor"]]:
    result = web_scraper_agent.invoke(state)
    return Command(
        update={"messages": [
            HumanMessage(content=result["messages"][-1].content, name="web_scraper")
        ]},
        goto="supervisor",
    )

Compile the research subgraph

research_supervisor = make_supervisor_node(llm, ["search", "web_scraper"])

research_builder = StateGraph(State)
research_builder.add_node("supervisor", research_supervisor)
research_builder.add_node("search", search_node)
research_builder.add_node("web_scraper", web_scraper_node)
research_builder.add_edge(START, "supervisor")

research_graph = research_builder.compile()

Writing Team Subgraph

The writing team produces the final deliverable. Three workers collaborate under their own mid-level supervisor:

WorkerToolsRole
note_taker create_outline, read_document Reads research output and creates a structured outline
doc_writer write_document, edit_document, read_document Writes and edits the full document from the outline
chart_generator read_document, python_repl_tool Generates charts via Python code execution

All document tools write to a shared temporary directory (WORKING_DIRECTORY): 

from pathlib import Path
from tempfile import TemporaryDirectory
from langchain_experimental.utilities import PythonREPL

_TEMP = TemporaryDirectory()
WORKING_DIRECTORY = Path(_TEMP.name)

@tool
def create_outline(
    points: Annotated[List[str], "List of main points or sections."],
    file_name: Annotated[str, "File path to save the outline."],
) -> str:
    """Create and save an outline."""
    with (WORKING_DIRECTORY / file_name).open("w") as file:
        for i, point in enumerate(points):
            file.write(f"{i + 1}. {point}\n")
    return f"Outline saved to {file_name}"

@tool
def write_document(
    content: Annotated[str, "Text content to be written into the document."],
    file_name: Annotated[str, "File path to save the document."],
) -> str:
    """Create and save a text document."""
    with (WORKING_DIRECTORY / file_name).open("w") as file:
        file.write(content)
    return f"Document saved to {file_name}"

@tool
def edit_document(
    file_name: Annotated[str, "Path of the document to be edited."],
    inserts: Annotated[Dict[int, str], "Line number to text mapping."],
) -> str:
    """Edit a document by inserting text at specific line numbers."""
    with (WORKING_DIRECTORY / file_name).open("r") as file:
        lines = file.readlines()
    for line_number, text in sorted(inserts.items()):
        lines.insert(line_number - 1, text + "\n")
    with (WORKING_DIRECTORY / file_name).open("w") as file:
        file.writelines(lines)
    return f"Document edited and saved to {file_name}"

@tool
def read_document(
    file_name: Annotated[str, "File path to read the document from."],
    start: Annotated[Optional[int], "The start line. Default is 0"] = None,
    end: Annotated[Optional[int], "The end line. Default is None"] = None,
) -> str:
    """Read the specified document."""
    with (WORKING_DIRECTORY / file_name).open("r") as file:
        lines = file.readlines()
    if start is None:
        start = 0
    return "\n".join(lines[start:end])

repl = PythonREPL()

@tool
def python_repl_tool(
    code: Annotated[str, "The python code to execute to generate your chart."],
):
    """Execute python code. Print values with print(...) to see output."""
    try:
        result = repl.run(code)
    except BaseException as e:
        return f"Failed to execute. Error: {repr(e)}"
    return f"Successfully executed:\n```python\n{code}\n```\nStdout: {result}"

Top-Level Graph — Composing Subgraphs

The top-level graph treats each compiled subgraph as a single node. When the top supervisor routes to research_team, the entire research subgraph runs to completion before control returns: 

def call_research_team(state: State) -> Command[Literal["supervisor"]]:
    response = research_graph.invoke({"messages": state["messages"][-1]})
    return Command(
        update={"messages": [
            HumanMessage(content=response["messages"][-1].content, name="research_team")
        ]},
        goto="supervisor",
    )

def call_paper_writing_team(state: State) -> Command[Literal["supervisor"]]:
    response = paper_writing_graph.invoke({"messages": state["messages"][-1]})
    return Command(
        update={"messages": [
            HumanMessage(content=response["messages"][-1].content, name="writing_team")
        ]},
        goto="supervisor",
    )

teams_supervisor = make_supervisor_node(llm, ["research_team", "writing_team"])

super_builder = StateGraph(State)
super_builder.add_node("supervisor", teams_supervisor)
super_builder.add_node("research_team", call_research_team)
super_builder.add_node("writing_team", call_paper_writing_team)
super_builder.add_edge(START, "supervisor")

super_graph = super_builder.compile()

This is the key architectural insight: a compiled graph is just a callable. research_graph.invoke(...) runs the full inner loop (mid-supervisor → search → supervisor → web_scraper → supervisor → FINISH) and returns the final message. The top supervisor sees one message from “research_team” and decides whether to send work to the writing team next.

Safety Note

The python_repl_tool executes arbitrary code locally via PythonREPL. The notebook includes an explicit warning: this is unsafe in production without a sandbox. Replace it with a containerized code runner or a chart-generation API before deploying.

Key Takeaways

Remember These Four

1. A compiled LangGraph is a callable. Subgraphs run independently and return a result message to the parent graph — this is how hierarchy works.

2. Reuse the supervisor factory. make_supervisor_node with structured output works at every level; only the members list changes.

3. Workers always report back. Every worker node ends with goto="supervisor" so the mid-level supervisor can pick the next step or FINISH.

4. Start flat, add hierarchy when needed. Don't jump to three levels on day one. Begin with a peer network or single supervisor; promote to nested teams when worker count or task complexity demands it.
