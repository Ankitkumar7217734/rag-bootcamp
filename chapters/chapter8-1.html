Query Expansion

The first query-enhancement technique — take a short or vague question and rewrite it into a richer, broader one so the retriever catches everything relevant, not just literal matches.

What is Query Expansion?

Users type short, imperfect queries: "langchain memory", "tools in LLM", "retrieval". Each is too thin for the retriever to find the best chunks. Query expansion uses an LLM to reformulate and broaden the query — adding synonyms, related phrases, and likely sub-topics — before it ever reaches the vector store.

The retriever then searches with a fuller, more descriptive query, so it catches documents that the original short query would have missed.

The Core Idea

One vague query in → one rich, expanded query out → more (and more relevant) chunks retrieved. Better queries → better retrieved chunks → better grounded LLM answers.

Why It Matters in RAG

Problem (original query)Solution (enhanced query)
"LangChain memory""LangChain memory modules, conversation memory"
"tools in LLM""LangChain tools, APIs, calculator, agent tools"
"retrieval""vector retrieval, dense search, BM25, MMR"

Each expansion turns a one- or two-word fragment into a phrase rich with the vocabulary that actually appears in the documents — dramatically improving the odds of a semantic match.

How Query Expansion Works

The flow is simple: the raw query is passed through an LLM + prompt chain that rewrites it. The enhanced query is then sent to the retriever, which searches the vector store and returns top-k documents, optionally re-ranked, before the LLM generates the final answer.

Query Expansion in LangChain

The expansion is just another prompt-chain: an LLM rewrites the query, and the result feeds the normal RAG chain. Here we load the dataset, build an MMR retriever (from Chapter 7), and add an expansion step in front.

from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings
from langchain_classic.document_loaders import TextLoader
from langchain_classic.text_splitter import RecursiveCharacterTextSplitter
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough

# Step 1: Load and chunk the document
loader = TextLoader("Langchain Crewai Dataset.txt")
raw_docs = loader.load()
splitter = RecursiveCharacterTextSplitter(chunk_size=300, chunk_overlap=50)
chunks = splitter.split_documents(raw_docs)

# Step 2: Vector store + MMR retriever
embeddings = OpenAIEmbeddings(model="text-embedding-3-small",
                              dimensions=1536, api_key=openai_api_key)
vectorstore = FAISS.from_documents(chunks, embeddings)
retriever = vectorstore.as_retriever(search_type="mmr", search_kwargs={"k": 3})

Now the expansion chain — a small prompt that asks the LLM to broaden the question:

# Step 4: Query expansion chain
query_expansion_prompt = ChatPromptTemplate.from_template(
    """
    You are a helpful query expansion tool.
    The user has asked the following question: {query}
    Please expand the question to cover more possible answers.
    """
)
query_expansion_chain = query_expansion_prompt | llm | StrOutputParser()

# Try it
query_expansion_chain.invoke({"query": "what is langchain"})

Finally, wire expansion in front of the RAG chain — expand the query first, then retrieve and answer with the expanded version:

# RAG answering prompt
answer_prompt = ChatPromptTemplate.from_template("""
Answer the question based on the context below
context:
{context}
question: {input}
""")

def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

# Step 5: Full RAG pipeline with query expansion
rag_chain = (
    {"context": retriever | format_docs, "input": RunnablePassthrough()}
    | answer_prompt
    | llm
    | StrOutputParser()
)

# Expand the query, then run retrieval + answering on the expanded version
expanded_query = query_expansion_chain.invoke({"query": "what is langchain"})
response = rag_chain.invoke(expanded_query)
print("✅ Answer:\n", response)
Watch the Original Question

A subtle gotcha: if you replace the user's question entirely with the expanded text, the LLM may lose track of what was actually asked. In practice, keep the original question for the final answering prompt and use the expanded version only for retrieval — expansion is a retrieval aid, not a replacement for the user's intent.

Key Takeaway

Query expansion rewrites a thin query into a richer one via an LLM + prompt chain, catching synonyms and related phrasing the retriever would otherwise miss. It's a single extra step bolted onto the front of any RAG chain.
