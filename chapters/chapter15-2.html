Self-Reflection

The LLM grades its own answer: “Is this clear, complete, and accurate?” If not, the graph loops back and tries again.

Overview

In 15.1, the graph ran in a straight line: plan, retrieve, answer, done. There was no check on whether the answer was actually good. In production, that's dangerous — the LLM will confidently answer from incomplete or irrelevant context and never tell you it guessed.

Self-reflection adds a quality gate. After generating an answer, a second LLM call acts as a critic: “Is this complete and correct?” If the verdict is NO, the graph loops back to retrieval and tries again. If the verdict is YES (or we've hit the attempt limit), the graph exits. This is the chapter's first conditional edge — the mechanism that turns a pipeline into a loop.

Where this fits

From 4-Selfreflection.ipynb, building block 2 of 5. The retriever and responder from basic RAG are unchanged; we add a reflector node and a routing function that decides whether to exit or retry. Next in 15.3: splitting multi-topic questions into sub-queries.

Follow the red dashed arrow on the right: when the reflector says NO, control jumps back to the retriever. The same question is searched again (we haven't changed the query yet — that improvement comes in 15.4). The green path down to done fires when the answer passes or we've already tried twice.

Step 1 — Setup & State

The knowledge base is sample_docs.txt — notes on agent loops and transformer systems. The ingestion is the same one-liner from earlier chapters. What's new is the state: three extra fields that track the reflection verdict and how many times we've tried.

Imports, LLM, and vector store

import os
from typing import List
from pydantic import BaseModel
from langchain_openai import OpenAIEmbeddings
from langchain_classic.schema import Document
from langchain_classic.text_splitter import RecursiveCharacterTextSplitter
from langchain_classic.vectorstores import FAISS
from langchain_community.document_loaders import TextLoader
from langchain.chat_models import init_chat_model
from langgraph.graph import StateGraph, END

llm = init_chat_model("openai:gpt-4o")

docs = TextLoader("sample_docs.txt").load()
chunks = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50).split_documents(docs)
vectorstore = FAISS.from_documents(chunks, OpenAIEmbeddings())
retriever = vectorstore.as_retriever()

The state schema

Everything the graph needs to carry between nodes lives here. The three new fields are what make self-reflection possible. 

class RAGReflectionState(BaseModel):
    question: str
    retrieved_docs: List[Document] = []
    answer: str = ""
    reflection: str = ""
    revised: bool = False
    attempts: int = 0
FieldSet byPurpose
reflectionreflect_on_answerThe critic's full text response (YES/NO + explanation).
revisedreflect_on_answerTrue when the answer failed the check — signals a retry is needed.
attemptsgenerate_answerIncremented each time the responder runs. Caps the loop at 2 tries.
Watch the naming

revised sounds like “already fixed,” but it actually means “needs revision” — it's True when the reflector says NO. The conditional edge checks not s.revised to decide whether to exit. Read the field as “retry needed” and the logic clicks.

Step 2 — The Nodes

Four node functions. The first two (retrieve, generate) are familiar from basic RAG. The third (reflect) is the new critic. The fourth (finalize) is a no-op exit node — LangGraph needs a named destination for the conditional edge to route to.

Node 1 — retrieve_docs

Unchanged from basic RAG. Fetch top-k chunks for the question.

def retrieve_docs(state: RAGReflectionState) -> RAGReflectionState:
    docs = retriever.invoke(state.question)
    return state.model_copy(update={"retrieved_docs": docs})

Node 2 — generate_answer

Format the retrieved context into a prompt, call the LLM, and store the answer. Critically, this node also increments attempts — that's how the loop knows when to stop even if the reflector keeps saying NO. 

def generate_answer(state: RAGReflectionState) -> RAGReflectionState:
    context = "\n\n".join([doc.page_content for doc in state.retrieved_docs])
    prompt = f"""
Use the following context to answer the question:

Context:
{context}

Question:
{state.question}
"""
    answer = llm.invoke(prompt).content.strip()
    return state.model_copy(update={"answer": answer, "attempts": state.attempts + 1})

Node 3 — reflect_on_answer (the critic)

A separate LLM call that reads the question and the draft answer, then returns a structured verdict. We ask the model to respond in a fixed format (Reflection: YES or Reflection: NO) so we can parse it reliably. If the response contains reflection: yes, the answer passed; otherwise we set revised = True to trigger a retry. 

def reflect_on_answer(state: RAGReflectionState) -> RAGReflectionState:
    prompt = f"""
Reflect on the following answer to see if it fully addresses the question.
State YES if it is complete and correct, or NO with an explanation.

Question: {state.question}

Answer: {state.answer}

Respond like:
Reflection: YES or NO
Explanation: ...
"""
    result = llm.invoke(prompt).content
    is_ok = "reflection: yes" in result.lower()
    return state.model_copy(update={"reflection": result, "revised": not is_ok})

Node 4 — finalize

Returns the state unchanged. It exists only as a named exit node the conditional edge can route to before hitting END. 

def finalize(state: RAGReflectionState) -> RAGReflectionState:
    return state

Step 3 — The Conditional Edge

This is the most important piece of the sub-chapter. After the reflector runs, a small function inspects the state and returns the name of the next node. LangGraph calls this a conditional edge — it's how you turn a straight pipeline into a loop without writing any explicit while logic.

Wiring the graph

builder = StateGraph(RAGReflectionState)

builder.add_node("retriever", retrieve_docs)
builder.add_node("responder", generate_answer)
builder.add_node("reflector", reflect_on_answer)
builder.add_node("done", finalize)

builder.set_entry_point("retriever")
builder.add_edge("retriever", "responder")
builder.add_edge("responder", "reflector")

So far, still a straight line: retriever → responder → reflector. The conditional edge after reflector is where the loop appears.

The routing function

builder.add_conditional_edges(
    "reflector",
    lambda s: "done" if not s.revised or s.attempts >= 2 else "retriever"
)

builder.add_edge("done", END)
graph = builder.compile()

Read the lambda carefully. It returns "done" (exit) when either:

  • not s.revised — the reflector said YES, answer is good enough.
  • s.attempts >= 2 — we've tried twice and must stop regardless.

Otherwise it returns "retriever", and the graph loops back to fetch documents again. The attempts >= 2 guard is non-negotiable in production — without it, a stubborn critic can spin forever.

Reflector saysAttemptsNext node
YES1done → END
NO1retriever (retry)
NO2done → END (cap hit)

Step 4 — Run It

Ask about transformer variants in production. On a well-indexed knowledge base, the first answer is usually good enough and the reflector passes it on the first try. 

user_query = "What are the transformer variants in production deployments?"
init_state = RAGReflectionState(question=user_query)
result = graph.invoke(init_state)

print("\n🧠 Final Answer:\n", result["answer"])
print("\n🔁 Reflection Log:\n", result["reflection"])
print("🔄 Total Attempts:", result["attempts"])

Print all three outputs. The reflection log shows you why the critic accepted or rejected the answer — useful when debugging a graph that keeps looping.

Output — Self-Graded Answer

Final Answer: Production transformer variants include BERT (classification, QA), the GPT series (generation, chat), RoBERTa, DistilBERT (smaller/faster), ALBERT (parameter-efficient), XLNet, T5 (text-to-text), and open-source GPT-Neo / GPT-J.

Reflection: Reflection: YES — the answer is complete and accurate. Total Attempts: 1 (no retry needed).

Key Takeaways

Remember These Three

1. A reflector node is a second LLM call that grades the first answer before the graph exits.

2. add_conditional_edges is what makes the graph autonomous — a function reads the state and returns the name of the next node.

3. Always cap retries with an attempts counter. Without it, a stubborn critic can loop forever.
