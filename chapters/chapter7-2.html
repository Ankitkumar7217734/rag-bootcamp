Re-Ranking Techniques

The second hybrid strategy — let a fast retriever pull a rough shortlist, then hand it to a slower, smarter model that re-scores and reorders so the truly best documents rise to the top.

A Second, Smarter Pass

A fast retriever (BM25, FAISS, or hybrid) is great at quickly pulling a rough shortlist — but its ordering is only approximate. Re-ranking is a second stage that takes that shortlist and re-scores it with a slower, more accurate model so the truly best documents float to the top.

  1. Retrieve fast: use BM25 / FAISS / hybrid to fetch the top-k documents quickly (e.g. k = 8).
  2. Re-score accurately: pass each query–document pair through a cross-encoder or an LLM that judges true relevance, then reorder.

The retriever optimises for speed over many documents; the reranker optimises for accuracy over a few. Using both gives you scale and precision.

Re-Ranking with an LLM

One practical approach: retrieve a generous top-k, then ask an LLM to rank them by relevance and return the order. Here the retriever pulls 8 candidates; the LLM returns their indices best-first, and we reorder.

from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

# 1. Fast retriever returns a generous shortlist
retriever = vectorstore.as_retriever(search_kwargs={"k": 8})
query = "How can I use LangChain to build an application with memory and tools?"
retrieved_docs = retriever.invoke(query)

# 2. Ask the LLM to rank the shortlist by relevance
prompt = ChatPromptTemplate.from_template("""
You are a helpful assistant. Rank the following documents from most to
least relevant to the user's question.

User Question: "{question}"

Documents:
{documents}

Return a comma-separated list of document indices, most relevant first
(e.g. 2,1,3,0).
""")

chain = prompt | llm | StrOutputParser()

doc_lines = [f"{i+1}. {doc.page_content}" for i, doc in enumerate(retrieved_docs)]
formatted_docs = "\n".join(doc_lines)
response = chain.invoke({"question": query, "documents": formatted_docs})

# 3. Parse the ranking and reorder the documents
indices = [int(x.strip()) - 1 for x in response.split(",") if x.strip().isdigit()]
reranked_docs = [retrieved_docs[i] for i in indices if 0 <= i < len(retrieved_docs)]

for i, doc in enumerate(reranked_docs, 1):
    print(f"\nRank {i}:\n{doc.page_content}")
Cross-Encoder vs LLM

A cross-encoder reads the query and a document together in one pass and outputs a single relevance score — far more accurate than comparing two separate embeddings, but too slow to run over a whole corpus. That's why it (or an LLM) runs only on the retriever's small shortlist. Bi-encoders (normal embeddings) are fast and used for the first stage; cross-encoders are precise and used for the second.

Key Takeaway

Re-ranking is a two-stage pattern: retrieve fast → re-score accurately. A cross-encoder or LLM judges the full query–document pair and reorders the shortlist, so the most relevant context reaches the LLM first — fewer hallucinations, better answers.
