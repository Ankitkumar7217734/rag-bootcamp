Chapter 7: Hybrid Search Strategies

Plain semantic search is good, not great. This chapter is about squeezing the best retrieval out of your data — by combining search methods, re-ordering results with a smarter model, and diversifying them so the LLM sees the whole picture, not the same fact five times.

Overview

Chapters 4–6 built a working retriever: embed the chunks, store the vectors, search by meaning. That gets you a long way — but production RAG systems rarely stop at a single similarity search. They layer on hybrid search strategies to lift both recall (finding everything relevant) and precision (ranking the best of it at the top).

This chapter splits into three sub-chapters, one per strategy, each solving a different weakness of plain vector search:

#StrategyProblem it solves
1Dense + Sparse RetrievalEmbeddings miss exact keywords; keyword search misses meaning. Combine both.
2Re-RankingA fast retriever's top-k is roughly relevant but poorly ordered. Re-score it with an accurate model.
3MMRTop results are often near-duplicates. Pick relevant and diverse documents.
The Core Idea

Each strategy is a different lever on the same goal: get the most relevant, most useful, least redundant context in front of the LLM. Better context in → better answer out. They're independent and stack together — many real pipelines use all three.

Explore Each Strategy

Work through the three sub-chapters in order — each is self-contained with its own diagram, worked example, and LangChain code:

  • 1 · Combining Dense & Sparse Retrieval →
    Sparse (BM25/TF-IDF, exact keywords) + dense (embeddings, meaning), the Score = α·dense + (1−α)·sparse formula, and LangChain's EnsembleRetriever.
  • 2 · Re-Ranking Techniques →
    The two-stage pattern: a fast retriever fetches a shortlist, then a slow, accurate cross-encoder or LLM re-scores and reorders it.
  • 3 · MMR — Maximal Marginal Relevance →
    Diversity-aware retrieval that balances relevance against redundancy with MMR(d) = λ·sim(d,q) − (1−λ)·max sim(d,s).

Putting It Together

These three strategies are not alternatives — they're layers. A strong production retrieval stage often chains all of them:

Hybrid retrieve (dense + sparse)   →   broad, high-recall shortlist
        ↓
Re-rank (cross-encoder / LLM)      →   precise ordering, noise removed
        ↓
MMR / diversity filter             →   drop near-duplicates
        ↓
Top-n context  →  LLM  →  grounded, well-rounded answer

Hybrid lifts recall, re-ranking lifts precision, and MMR lifts coverage — each addressing a different failure mode of plain vector search.

Key Takeaways

Remember These Four

1. Hybrid retrieval blends dense (embeddings, meaning) and sparse (BM25/TF-IDF, exact keywords) with Score = α·dense + (1−α)·sparse — LangChain's EnsembleRetriever does it with a weight list.

2. Re-ranking is a two-stage pattern: a fast retriever fetches a top-k shortlist, then a slow, accurate cross-encoder or LLM re-scores and reorders it so the best documents land on top.

3. MMR balances relevance and diversity via MMR(d) = λ·sim(d,q) − (1−λ)·max sim(d,s), dropping near-duplicates — use search_type="mmr" with lambda_mult as the λ dial.

4. They stack: hybrid boosts recall, re-ranking boosts precision, MMR boosts coverage — combine them for the strongest, least-redundant context into the LLM.
