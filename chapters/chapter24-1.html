Chatbot Evaluation

Learn the LangSmith experiment loop on a simple chatbot first — curated examples, an LLM correctness judge, a deterministic concision check, then a fair model-vs-model comparison.

Overview

Before evaluating retrieval, the notebook builds a five-example chatbot dataset and runs the same target twice: once with GPT-4o-mini and once with GPT-4 Turbo. The only changed variable is the model. That isolates the experiment machinery you will reuse for RAG in the next sub-chapter.

The Core Idea

Pair an LLM-as-a-judge for semantic correctness with a deterministic concision rule. One catches paraphrase; the other catches verbosity. Neither alone is enough.

Create the Five-Example Dataset

from langsmith import Client

client = Client()

dataset_name = "Chatbots Evaluation"
dataset = client.create_dataset(dataset_name)

client.create_examples(
    dataset_id=dataset.id,
    examples=[
        {
            "inputs": {"question": "What is LangChain?"},
            "outputs": {"answer": "A framework for building LLM applications"},
        },
        {
            "inputs": {"question": "What is LangSmith?"},
            "outputs": {
                "answer": "A platform for observing and evaluating LLM applications"
            },
        },
        {
            "inputs": {"question": "What is OpenAI?"},
            "outputs": {"answer": "A company that creates Large Language Models"},
        },
        {
            "inputs": {"question": "What is Google?"},
            "outputs": {"answer": "A technology company known for search"},
        },
        {
            "inputs": {"question": "What is Mistral?"},
            "outputs": {"answer": "A company that creates Large Language Models"},
        },
    ],
)

Each reference is intentionally short because the target is instructed to answer in one sentence. References should specify the essential facts, not force the model to reproduce one exact wording.

Evaluator 1: Correctness With an LLM Judge

Exact string equality would mark a correct paraphrase as wrong. The notebook instead gives a judge the question, ground truth, and predicted answer, then constrains its final decision to CORRECT or INCORRECT. 

import openai
from langsmith import wrappers

openai_client = wrappers.wrap_openai(openai.OpenAI())

eval_instructions = (
    "You are an expert professor specialized in grading students' "
    "answers to questions."
)

def correctness(inputs: dict, outputs: dict, reference_outputs: dict) -> bool:
    user_content = f"""You are grading the following question:
{inputs['question']}
Here is the real answer:
{reference_outputs['answer']}
You are grading the following predicted answer:
{outputs['response']}
Respond with CORRECT or INCORRECT:
Grade:
"""
    response = openai_client.chat.completions.create(
        model="gpt-4o-mini",
        temperature=0,
        messages=[
            {"role": "system", "content": eval_instructions},
            {"role": "user", "content": user_content},
        ],
    ).choices[0].message.content

    return response == "CORRECT"

Evaluator 2: Deterministic Concision

Not every metric needs another model call. The notebook defines a fast, transparent rule: the generated response must be shorter than twice the reference answer's character length. 

def concision(outputs: dict, reference_outputs: dict) -> bool:
    return len(outputs["response"]) < 2 * len(reference_outputs["answer"])

LLM-as-a-judgeUnderstands paraphrase and meaningCosts money; can drift or disagreeSemantic correctness

Run and Compare Chatbot Experiments

The target adapter is deliberately small. LangSmith passes one example's inputs; the target calls the application and returns a dictionary whose keys match what the evaluators expect. 

default_instructions = (
    "Respond to the users question in a short, concise manner "
    "(one short sentence)."
)

def my_app(
    question: str,
    model: str = "gpt-4o-mini",
    instructions: str = default_instructions,
) -> str:
    return openai_client.chat.completions.create(
        model=model,
        temperature=0,
        messages=[
            {"role": "system", "content": instructions},
            {"role": "user", "content": question},
        ],
    ).choices[0].message.content

def ls_target(inputs: dict) -> dict:
    return {"response": my_app(inputs["question"])}
mini_results = client.evaluate(
    ls_target,
    data=dataset_name,
    evaluators=[correctness, concision],
    experiment_prefix="openai-4o-mini-chatbot",
)

The source run evaluated all five examples in about ten seconds. It then redefined the target to use GPT-4 Turbo and launched a second experiment against the same dataset and evaluators. 

def ls_target(inputs: dict) -> dict:
    return {
        "response": my_app(
            inputs["question"],
            model="gpt-4-turbo",
        )
    }

turbo_results = client.evaluate(
    ls_target,
    data=dataset_name,
    evaluators=[correctness, concision],
    experiment_prefix="openai-4-turbo-chatbot",
)
Fair Model Comparison

Both models receive the same five questions, references, instructions, temperature, and rubrics. The experiment name records the changed variable. Compare aggregate scores, but inspect row-level failures too: two models can have the same average while failing on different questions.

Key Takeaways

Remember These Four

1. Freeze a named dataset of inputs and reference outputs so every model faces the same cases.

2. An LLM judge with temperature 0 and a CORRECT/INCORRECT decision handles paraphrase better than string equality.

3. Add a deterministic metric (concision by length ratio here) for cheap, reproducible checks.

4. Compare models by changing one variable and inspecting both aggregate scores and row-level failures.
